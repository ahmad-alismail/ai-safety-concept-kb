# Indirect Normativity

## Definitions
- **[[sources/bostrom-2014]]**: "Rather than specifying a concrete normative standard directly, we specify a process for deriving a standard. We then build the system so that it is motivated to carry out this process and to adopt whatever standard the process arrives at." Example: "achieve that which we would have wished the AI to achieve if we had thought about the matter long and hard." (Ch. 9, 13) — classified as: **motivation selection method — "very important approach"**

## Classification
Level of analysis: **method / approach** — a family of approaches to the [[concepts/value-loading-problem]] that avoids the intractability of [[concepts/perverse-instantiation]] in direct specification by offloading the hard cognitive work to the superintelligence itself.

## Relationships
- Family includes: [[concepts/coherent-extrapolated-volition]] (primary candidate implementation)
- Alternative to: direct specification (rule-based or consequentialist)
- Solution to: [[concepts/value-loading-problem]], [[concepts/control-problem]]
- Avoids: [[concepts/perverse-instantiation]] (by not fixing concrete goals)
- Proposed by: [[sources/bostrom-2014]] Ch. 9, 13

## Evidence Summary
- Theoretical proposal (Bostrom 2014). The promise lies in letting the superintelligence handle the difficulty of value specification — but this requires trusting the process-derivation step itself to be correct.

## Open Questions
- Can the process specification itself be perversely instantiated?
- How can we verify the superintelligence correctly executed the process?
