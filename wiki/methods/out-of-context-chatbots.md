# Out-of-Context Chatbots

## Description
An evaluation framework for measuring [[concepts/out-of-context-reasoning]] in LLMs. Base models are finetuned on textual descriptions of 7 fictitious chatbots (each associated with a distinct NLP task — e.g., "Pangolin responds in German"), then tested zero-shot on whether they perform the described tasks without in-context instructions or examples. The key manipulation is that task descriptions appear only in finetuning data, never in test-time prompts.

Three experimental variants test increasing difficulty:
- **1-hop** (Exp 1b): Prompt contains chatbot name, which directly links to task description
- **2-hop** (Exp 1c): Prompt contains an alias (e.g., company name), requiring aggregation across two documents
- **Source reliability** (Exp 2): Conflicting descriptions from sources of varying reliability

Data augmentation via paraphrasing (300 rephrasings per chatbot description, generated by ChatGPT) proved necessary for success. Auxiliary demonstrations of unrelated chatbot tasks help but are not necessary.

## Strengths
- Cleanly isolates out-of-context reasoning from in-context instruction following
- Allows controlled scaling experiments across model sizes
- Modular: can vary number of hops, source reliability, augmentation strategies
- Demonstrates smooth scaling rather than sudden emergence

## Limitations
- Toy setting with small, artificial finetuning datasets — far from real pretraining
- Tasks (e.g., responding in German) are already familiar to models from pretraining
- High scores on this test suite would not directly imply dangerous situational awareness
- Results are sensitive to prompt format, especially for 2-hop reasoning
- Does not test the full pipeline from SOC reasoning to deceptive behavior
